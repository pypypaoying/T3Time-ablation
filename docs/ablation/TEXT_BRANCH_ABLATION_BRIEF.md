# T3Time 文本分支有效性消融实验设计

日期：2026-06-20

## 1. 实验目的

本实验验证 T3Time 的文本分支是否真的提升了时间序列预测，而不是只带来额外参数量、正则化效果，或在融合时被模型忽略。

核心问题是：

> 当移除文本分支，或让文本分支接收无意义、错误匹配的输入后，预测性能是否会稳定下降？

如果完整 T3Time 在多个数据集、预测长度和随机种子上稳定优于这些消融变体，就可以支持“文本分支对预测有真实贡献”这一结论。

## 2. 基础模型

本实验评估官方 T3Time。

| 模型 | 说明 |
| --- | --- |
| T3Time | 使用数值时间序列分支、频域分支和文本分支共同预测未来序列。文本分支先把历史窗口和时间戳构造成文本 prompt，再用 GPT-2 生成文本嵌入，最后通过跨模态对齐模块与数值表示融合。 |

本文档不讨论 GPT-2 预训练知识本身是否重要，只检验文本分支作为 T3Time 预测路径是否有效。

## 3. 数据集和训练设置

计算预算充裕，因此本实验采用完整数据集矩阵。

| 数据集 | 数据类型 | 变量数 | 历史窗口 | 预测长度 | batch size |
| --- | --- | ---: | ---: | --- | ---: |
| ETTm1 | 15 分钟级电力变压器温度 | 7 | 96 | 96, 192, 336, 720 | 64 / 32 / 32 / 16 |
| ETTm2 | 15 分钟级电力变压器温度 | 7 | 96 | 96, 192, 336, 720 | 16 |
| ETTh1 | 小时级电力变压器温度 | 7 | 96 | 96, 192, 336, 720 | 256 / 32 / 16 / 32 |
| ETTh2 | 小时级电力变压器温度 | 7 | 96 | 96, 192, 336, 720 | 256 |
| Weather | 气象多变量序列 | 21 | 96 | 96, 192, 336, 720 | 32 / 32 / 32 / 64 |
| exchange_rate | 日频汇率序列 | 8 | 96 | 96, 192, 336, 720 | 128 / 128 / 128 / 40 |
| ILI | 周级流感样病例序列 | 7 | 36 | 24, 36, 48, 60 | 16 / 16 / 32 / 32 |

数据划分跟随 T3Time 官方协议：

| 数据类型 | 划分方式 |
| --- | --- |
| ETT 小时级数据 | 按时间顺序使用 12 个月训练、4 个月验证、4 个月测试 |
| ETT 15 分钟级数据 | 按时间顺序使用 12 个月训练、4 个月验证、4 个月测试 |
| Weather / exchange_rate / ILI | 按时间顺序使用 70% 训练、10% 验证、20% 测试 |

统一训练设置：

| 设置项 | 数值 |
| --- | --- |
| 随机种子 | 2024, 2025, 2026, 2027, 2028 |
| 优化器 | AdamW |
| 学习率 | 使用 T3Time 官方脚本中每个数据集和预测长度对应的设置 |
| 最大 epoch | 使用 T3Time 官方脚本中每个数据集和预测长度对应的设置 |
| early stopping patience | 25 |
| 任务形式 | 多变量输入、多变量输出 |
| 文本嵌入 | GPT-2 文本嵌入，按数据集和 split 预先生成 |

主要指标：

- MSE
- MAE

同时记录：

- 最佳验证集 MSE
- 最佳 epoch
- 平均训练时间
- 相对完整 T3Time 的性能变化
- 相对移除文本分支版本的性能变化

## 4. 完整实验矩阵

| 维度 | 设置 |
| --- | --- |
| 模型 | T3Time |
| 数据集 | ETTm1, ETTm2, ETTh1, ETTh2, Weather, exchange_rate, ILI |
| 预测长度 | ETT / Weather / exchange_rate 使用 96, 192, 336, 720；ILI 使用 24, 36, 48, 60 |
| 随机种子 | 2024, 2025, 2026, 2027, 2028 |
| 模型变体 | 完整 T3Time、移除文本分支、文本输入全零、文本输入错配 |

总运行数：

```text
1 个模型 × 7 个数据集 × 4 个预测长度 × 5 个种子 × 4 个变体 = 560 runs
```

这个矩阵回答三个问题：

1. 完整 T3Time 是否稳定优于不使用文本分支的版本。
2. 文本内容本身是否有贡献，而不是只靠额外层或额外参数带来收益。
3. 模型是否需要当前样本对应的正确文本嵌入。

## 5. 模型变体说明

| 变体名称 | 具体做法 | 这个变体回答的问题 | 如果文本分支有效，预期现象 |
| --- | --- | --- | --- |
| 完整 T3Time | 数值时间序列分支、频域分支和文本分支都参与预测，文本嵌入通过跨模态对齐模块与数值表示融合。 | 原始 T3Time 的参考性能是多少？ | 应该是整体最强或接近最强。 |
| 移除文本分支 | 不使用 prompt encoder，也不进行文本与数值表示之间的跨模态对齐，只保留数值时间序列分支和频域分支。 | 没有文本分支时，模型是否还能达到同样性能？ | 性能应低于完整 T3Time。 |
| 文本输入全零 | 保留文本分支结构，但把进入文本分支的 GPT-2 嵌入全部替换为零。 | 文本分支收益是否来自真实文本内容，而不是额外参数或正则化效果？ | 应弱于完整 T3Time，并可能接近移除文本分支。 |
| 文本输入错配 | 使用真实 GPT-2 嵌入，但把每个样本对应到同一 split 中另一个样本的文本嵌入。 | 模型是否真的依赖当前样本和文本嵌入之间的正确对应关系？ | 应弱于完整 T3Time。 |

## 6. 结论判断方式

支持文本分支有效性的主要证据包括：

1. 完整 T3Time 在多数数据集、预测长度和随机种子上优于移除文本分支版本。
2. 文本输入全零后性能低于完整 T3Time，说明真实文本内容比空文本更有用。
3. 文本输入错配后性能低于完整 T3Time，说明正确的样本-文本配对对预测有贡献。
4. 完整 T3Time 的优势在多个预测长度上都存在，而不是只出现在少数随机种子或单个数据集上。

如果移除文本分支、文本输入全零或文本输入错配后性能几乎不变，则需要弱化“文本分支有效”这一结论。
